# 场景层 — 说明文档

> 前四层是通用底座，第五层是"博主皮肤"——把上面所有层组合成内容生产的流水线。
> 不同场景可以复用同一套知识库、Skills、工作流，只是组合方式不同。

---

## 核心原则

1. **场景 = 特定目标 + 资源组合**：每个场景有明确的目标，组合使用下四层的资源
2. **场景可叠加**：一个人可以同时有多个场景（博主、创业者、学习者...）
3. **场景会演化**：随着需求变化，场景会新增、合并、或淘汰

---

## 当前场景

```
05-scenes/
├── content-production/   ← 内容生产（博主核心场景）
└── learning/             ← 持续学习（输入→输出的闭环）
```

---

## 场景文件格式

每个场景包含：
- **场景定义**：这个场景是什么、目标是什么
- **资源映射**：这个场景用到哪些身份/知识/工具/工作流
- **典型流程**：从触发到完成的完整流程
- **度量指标**：怎么判断这个场景运转良好
