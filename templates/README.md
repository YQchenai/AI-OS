# AI-OS — 个人 AI 操作系统

> **版本**: v1.0.0
> **定价**: 99-299 元
> **目标客户**: 内容创作者/博主（优先），后续扩展至创业者、产品经理等

---

## 产品概述

AI-OS 是一套由 Markdown 文档组成的**个人 AI 协作框架**。它不是某个具体的软件或 App，而是一套经过验证的结构化方法论，让任何 AI 工具（SOLO、Claude Code、ChatGPT 等）能够真正"认识"用户、"理解"用户、并高效地"替"用户干活。

### 核心卖点

1. **工具无关** — 纯 Markdown，任何 AI 工具都能用，随时可迁移
2. **开箱即用** — 运行初始化脚本，回答几个问题，30 分钟完成搭建
3. **越用越聪明** — 知识库持续积累，AI 对你的理解越来越深
4. **即战力提升** — 预置 5 个 Skills + 3 条工作流，第一天就能用

## 产品交付物

### 1. 初始化脚本 `init.sh`

客户在 SOLO（或任何支持 Bash 的环境）中运行，通过交互式问答：
- 收集基础信息（身份、领域、平台、风格）
- 收集深度偏好（语调、排版、AI 禁忌）
- 自动创建五层架构目录
- 自动生成个性化配置文件（CLAUDE.md、memory 文件等）
- 自动生成预置 Skills 和工作流

**交互流程**：
```
Step 0: 确认安装路径
Step 1: 基础信息收集（你是谁、做什么、在哪些平台）
Step 2: 深度偏好设置（风格、语调、AI 禁忌）
Step 3: 创建五层架构目录
Step 4: 生成个性化身份层文件
Step 5: 生成知识层模板
Step 6: 生成工具层 Skills（5个预置）
Step 7: 生成协作层工作流（3条预置）
Step 8: 生成场景层配置
Step 9: 生成使用指南
```

### 2. 通用模板文件包

作为初始化脚本的备用方案，客户也可以直接使用预配置的模板文件：
- 所有模板文件已预填示例内容
- 客户只需替换 `[占位符]` 即可

### 3. 使用指南

- 详细的分步搭建指南
- 日常使用说明
- 维护清单
- 常见问题解答

## 五层架构

```
ai-os/
├── 01-identity/          身份层 — AI 认识你的"名片"
│   ├── CLAUDE.md         全局配置（AI 入口）
│   └── memory/           记忆文件（画像、风格、偏好、项目）
│
├── 02-knowledge/         知识层 — 你的知识"燃料库"
│   ├── identity/         定位、风格
│   ├── audience/         读者画像
│   ├── content/          内容归档（按平台）
│   ├── materials/        素材库
│   └── decisions/        决策记录
│
├── 03-tools/             工具层 — AI 的"肌肉记忆"
│   └── skills/
│       ├── global/       通用 Skills（去AI腔、标题生成、多平台改编）
│       ├── xiaohongshu/  小红书 Skills
│       ├── wechat/       公众号 Skills
│       └── utilities/    辅助 Skills
│
├── 04-workflows/         协作层 — 串联技能的"剧本"
│   ├── idea-to-publish   从想法到发布
│   ├── repurpose         一鱼多吃
│   └── content-review    内容复盘
│
└── 05-scenes/            场景层 — 垂直输出"管线"
    ├── content-production  内容生产
    └── learning            持续学习
```

## 客户使用流程

### 方式一：运行初始化脚本（推荐）

```bash
# 在 SOLO 中运行
cd /path/to/AI-OS
bash init.sh
# 回答几个问题 → 自动完成所有配置
```

### 方式二：手动使用模板

1. 将 `templates/` 文件夹复制到工作目录
2. 按照使用指南逐个替换占位符
3. 开始使用
