# 协作层 — 说明文档

> Skill 是单个能力，工作流是把 Skills 串起来的**剧本**。
> 之前要 5 个 App 切换、5 次复制粘贴，现在一条工作流搞定。

---

## 核心原则

1. **先跑通一条，再复制思路**：不要一开始就设计 10 条工作流
2. **每条工作流解决一个高频场景**：问自己"这个流程我每周至少用 3 次吗？"
3. **明确人机分工**：每一步标注是 AI 做、人做、还是人机协作
4. **记录实际耗时**：优化前后对比，量化效率提升

---

## 工作流文件格式

每个工作流文件包含：
- **名称**：这个工作流做什么
- **触发条件**：什么时候启动
- **输入**：需要什么
- **输出**：产出什么
- **步骤链**：每一步用什么 Skill / 做什么 / 谁负责
- **预计耗时**：从输入到输出的总时间
- **优化记录**：每次改进的记录

---

## 推荐优先创建的工作流

### 第一优先级
1. **从想法到小红书发布** — 最核心的内容生产流程
2. **从想法到公众号发布** — 深度内容生产流程

### 第二优先级
3. **一鱼多吃** — 一篇内容改编为多平台版本
4. **内容复盘** — 发布后的数据分析与知识沉淀

### 第三优先级
5. **选题雷达** — 定期扫描热门话题和竞品动态
6. **素材收集** — 日常碎片化素材的整理归档

---

## 快速启动

1. 选你**最高频**的一个内容生产场景
2. 把你现在的步骤写下来
3. 每一步问：这步 AI 能替我做到第几成？
   - 能替 80% 以上 → 变成 Skill
   - 能替 50% → "AI 起草 + 人校准"
   - 替不了 → 留给人
4. 把步骤串成工作流
5. 跑一次，记录问题
6. 优化，再跑
