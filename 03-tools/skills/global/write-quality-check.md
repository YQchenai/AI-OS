# 写作质量自检

> 版本：v1.0 | 更新：2026-04-24

## 触发条件

- 公众号长文写完正文后，在去 AI 腔之后自动调用
- 任何需要高质量输出的写作任务，作为最终质检环节

## 输入

- 已完成去 AI 腔的正文
- `01-identity/memory/writing_style.md` 风格标本

## 输出

- 质检报告（逐层评分 + 具体问题 + 修复建议）
- 修复后的最终版正文

## 执行步骤

---

### L1 硬性规则检查（必须全部通过）

逐条扫描，发现一条修一条，不允许跳过。任何一条不通过都必须修复。

#### 1.1 禁用词复查

对照 `de-ai-tone` 的禁用词表，全文搜索以下词汇，发现即替换：

| 禁用 | 替换为 |
|------|--------|
| 说白了 | 坦率的讲、其实就是 |
| 意味着什么 / 这意味着 | 那结果会怎样呢、所以呢 |
| 本质上 | 说到底、其实 |
| 换句话说 | 你想想看、也就是说 |
| 不可否认 | 直接删掉 |
| 综上所述 / 总的来说 | 换成具体的回扣句 |
| 首先...其次...最后 | 用自然转场词替代 |
| 值得注意的是 / 不难发现 | 删掉，直接说 |
| 让我们一起 | 咱们来 |
| 在当今社会 | 删掉，直接进入主题 |
| 随着XX的发展 | 删掉，直接说结论 |
| 旨在 | 为了 |
| 赋能 | 帮助/支持 |
| 颗粒度 | 细节 |
| 闭环 | 完整流程 |
| 打法 | 方法/策略 |
| 沉淀 | 积累/总结 |

**通过标准**：0 个禁用词残留。

#### 1.2 标点符号检查

| 禁用 | 处理方式 |
|------|----------|
| 冒号「：」 | 用逗号代替 |
| 破折号「——」 | 用逗号或句号 |
| 双引号「""」「"」」 | 用「」或直接不加 |

**通过标准**：0 个禁用标点残留。

#### 1.3 结构性套话扫描

- 「让我们来看看」「接下来让我们」→ 删掉或换成口语化转场
- 「在当今...的时代」「随着...的发展」→ 删掉，直接进入主题
- 连续 bullet point 罗列超过 3 个 → 改成自然段落叙述
- 大段加粗超过 2 行 → 拆开，只加粗关键词

**通过标准**：0 个结构性套话残留。

#### 1.4 空泛工具名检查

- 「AI工具」「某个模型」「一款软件」→ 必须写出具体名字
- 「某个平台」「某个网站」→ 写出具体名称

**通过标准**：所有工具/平台/软件都有具体名称。

#### 1.5 假设性例子检查

- 「比如有一次...」「举个例子...」后面跟着明显编造的场景 → 删掉或换成真实细节
- AI 生成的"虚构经历"→ 必须替换为用户提供的真实素材

**通过标准**：0 个编造的假设性例子。

---

### L2 风格一致性检查

L1 通过后，检查文章风格是否统一、节奏是否自然。

#### 2.1 开头检查

- [ ] 是否从具体事件/场景切入（不是宏大叙事）
- [ ] 前 200 字是否有足够的"钩子"让人想继续读
- [ ] 开头语气是否与全文一致

**不通过时修复**：回到开头，用以下技巧之一重写：
- 叙事启动：「故事是这样的。」
- 荒诞事实：直接抛出一个让人「？？？」的事实
- 热点破题：「最近这两天，被一个xxx给刷屏了。」
- 好奇心驱动：「这两天在网上刷到了一张图，很有意思。」

#### 2.2 节奏检查

- [ ] 长短句是否交替出现（不能全是长句或全是短句）
- [ ] 是否至少有 3 处一句话独立成段
- [ ] 是否用逗号制造了口语化停顿
- [ ] 偏离主线后是否有"扣主线句"拉回来

**不通过时修复**：
- 长句拆短，短句合并，制造节奏变化
- 在关键观点处插入一句话独立成段
- 在转场处加入逗号停顿

#### 2.3 口语化程度检查

- [ ] 全文是否包含 8-10 个口语化表达
- [ ] 口语化表达是否自然融入（不是生硬插入）
- [ ] 是否至少有一处自嘲或承认不足

**不通过时修复**：从以下词组库中选取合适的表达嵌入：

**转场过渡**：坦率的讲、说真的、我是真的觉得、反正我觉得、怎么说呢、其实吧、你想想看、回到xxx这块、说到这里、话又说回来

**表达判断**：我有时候觉得、我一直觉得、不是说xxx不行而是说、我自己的感受是、以我的经验来看、至少对我来说

**承认自嘲**：说实话我也不确定、我自己也还在摸索、可能有些想法还不成熟、愚钝如我、我也不敢说一定对、这个我自己也踩过坑

**情绪表达**：这种感觉太爽了、我当时就愣住了、太离谱了、给我一下子整不会了、说实话有点上头、看完真的沉默了

**拉近距离**：很多朋友可能不知道、可能有小伙伴纳闷、大家也都知道、你们应该也有这种感觉、不知道你们有没有遇到过

#### 2.4 标点符号二次确认

- [ ] 全文无冒号
- [ ] 全文无破折号
- [ ] 全文无双引号（需要引用时用「」）

**通过标准**：全部符合。

---

### L3 内容质量检查

L2 通过后，检查文章的内容深度和价值。

#### 3.1 观点支撑度

- [ ] 每个核心观点是否至少有一个具体场景/人物/对话/数据支撑
- [ ] 论证逻辑是否自洽（没有前后矛盾）
- [ ] 是否有"观点+证据+感受"的完整结构

**不通过时修复**：
- 空观点补充具体案例或数据
- 逻辑矛盾处重新梳理因果关系
- 纯论述处加入个人感受或场景描写

#### 3.2 知识输出质量

- [ ] 知识是否以"聊着聊着顺手掏出来"的方式呈现（不是"下面我来科普"）
- [ ] 引用是否自然融入论述（像"聊着聊着想起来的"）
- [ ] 是否有至少一处让读者"学到了"的信息增量

**不通过时修复**：
- 生硬的科普段落改写成对话式叙述
- 独立的引用融入上下文
- 补充至少一个读者可能不知道的信息点

#### 3.3 文化升维度

- [ ] 文章是否从具体事件连接到了更大的文化/哲学/历史参照物
- [ ] 升维是否自然（不是硬凑）
- [ ] 结尾是否有余韵（不是戛然而止也不是拖泥带水）

**不通过时修复**：
- 在核心论述后加入一个"跳出来看"的视角
- 找一个与主题相关的更大参照物（历史事件、文化现象、哲学概念）
- 结尾选择合适的收尾方式：引用收尾 / 哲思余韵 / 行动呼吁 / 信念宣言 / 回环呼应

#### 3.4 同理心检查

- [ ] 是否站在读者角度思考过"这跟我有什么关系"
- [ ] 涉及对立观点时，是否先理解对方再切入自己视角
- [ ] 是否用「我也面临这个问题」连接个人经历和公共议题

**不通过时修复**：
- 在关键论述前加入"你可能也有这种感觉"之类的连接句
- 对立观点处补充"我理解为什么有人会这么想，但是..."
- 加入一段"我自己也遇到过类似情况"的经历

---

### L4 活人感终审

L3 通过后，做最终的"活人感"判断。这一层没有硬性标准，靠整体感觉。

#### 4.1 温度感

读完全文，问自己：这篇文章有温度吗？像是一个真人在跟你聊天，还是一篇精心包装的文案？

**判断标准**：
- ✅ 有温度：读完后能感受到作者的个性、情绪和态度
- ❌ 没温度：信息完整但感受不到"人"的存在

**不通过时修复**：
- 加入更多"我"的视角和感受
- 在关键处加入情绪化的表达（不是煽情，是真实反应）
- 放松一些，允许出现"不完美"的表达

#### 4.2 独特性

这篇文章换个博主也能写出来吗？还是只有这个人能写？

**判断标准**：
- ✅ 有独特性：包含只有作者才有的经历、观点组合或表达方式
- ❌ 没独特性：把作者名字换成任何人都不违和

**不通过时修复**：
- 加入只有作者才有的真实经历或观察
- 用作者独特的表达方式重写某些段落
- 加入作者特有的"口头禅"或表达习惯

#### 4.3 姿态感

作者的姿态是什么？是居高临下的"老师"，还是平起平坐的"朋友"？

**判断标准**：
- ✅ 好姿态：像朋友分享发现，不是专家在授课
- ❌ 差姿态：说教感重、全知全能、没有软肋

**不通过时修复**：
- 加入自嘲或承认不足
- 把"你应该"换成"我发现...你也可以试试"
- 减少"权威断言"，增加"个人感受"

#### 4.4 流畅度

出声读一遍全文，哪里磕巴？

**判断标准**：
- ✅ 流畅：读起来像说话一样自然
- ❌ 不流畅：有拗口的长句、生硬的转折、读不下去的段落

**不通过时修复**：
- 拗口的长句拆成短句
- 生硬的转折加入过渡句
- 读不下去的段落考虑删除或重写

## 质检报告模板

完成四层检查后，输出以下格式的质检报告：

```
## 质检报告

### L1 硬性规则
- 禁用词：[通过/发现X处，已修复]
- 标点符号：[通过/发现X处，已修复]
- 结构性套话：[通过/发现X处，已修复]
- 空泛工具名：[通过/发现X处，已修复]
- 假设性例子：[通过/发现X处，已修复]

### L2 风格一致性
- 开头：[通过/已重写]
- 节奏：[通过/已调整]
- 口语化：[通过/已补充X处]
- 标点二次确认：[通过]

### L3 内容质量
- 观点支撑度：[通过/已补充X处]
- 知识输出：[通过/已调整]
- 文化升维：[通过/已补充]
- 同理心：[通过/已补充]

### L4 活人感终审
- 温度感：[通过/已加强]
- 独特性：[通过/已补充]
- 姿态感：[通过/已调整]
- 流畅度：[通过/已优化]

### 总结
- 总体评级：[A/B/C/D]
- 主要问题：[一句话总结]
- 修复内容：[列出所有修改点]
```

### 评级标准

- **A**：四层全部通过，文章可以直接发布
- **B**：L1-L3 通过，L4 有小问题但已修复，可以发布
- **C**：L1-L2 通过，L3 有问题需人工确认
- **D**：L1 未通过，必须修复后重新质检

## 注意事项

- 四层检查必须按顺序执行，不能跳层
- L1 是硬性底线，任何一条不通过都必须修复
- L2-L4 的判断需要参考 `writing_style.md` 中的风格标本来校准
- 质检报告必须如实输出，不能跳过任何检查项
- 如果评级为 C 或 D，建议用户参与修改后再重新质检
